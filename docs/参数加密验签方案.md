# 参数加密与验签方案

## 一、设计原则

- **安全底线**：任何外部回调必须验签；敏感接口可启用参数加密
- **密钥管理**：密钥通过配置/环境变量注入，禁止硬编码
- **可插拔**：验签中间件可灵活应用于指定路由组

## 二、请求验签

### 2.1 签名算法

```
签名字符串 = key1=value1&key2=value2&key={secret}
签名结果   = SHA256(签名字符串).ToUpper()
```

- 参与签名的参数：**所有业务参数** + `timestamp` + `nonce`，按 key 字典序排序
- 排除：`sign`、`signature` 及空值参数

### 2.2 请求头

| Header       | 必填 | 说明                    |
|-------------|------|-------------------------|
| X-Signature | 是   | 签名字符串（Hex 大写）  |
| X-Timestamp | 是   | 时间戳，Unix 秒         |
| X-Nonce     | 是   | 随机串，防重放          |

### 2.3 参数来源

- GET：Query 参数
- POST 表单：Form 参数
- POST JSON：Body 顶层简单字段（string/number/bool）

### 2.4 防重放

- **时间窗口**：timestamp 与服务器时间差 ±5 分钟
- **Nonce 唯一**：Redis SETNX(nonce_key, 1, 10s)，重复 nonce 拒绝

## 三、参数加密（可选）

适用于敏感接口（提现、支付等）。

### 3.1 加密算法

- AES-256-GCM（认证加密，防篡改）

### 3.2 请求格式

- Header：`X-Encrypted: true`
- Body：`Base64(AES-GCM(JSON 明文))`

### 3.3 客户端流程

1. 构造业务参数 JSON
2. 对 JSON 字符串 AES-GCM 加密
3. Base64 编码后放入 Body
4. 对 **明文参数** 计算签名，放入 X-Signature（timestamp、nonce 在 Header）

### 3.4 服务端流程

1. 若 X-Encrypted=true，先解密 Body 得到明文
2. 合并参数（Query + Form + 解密后的 Body）
3. 校验 timestamp、nonce
4. 校验签名

## 四、使用示例

### 客户端签名生成（Go）

```go
import "fire-mirage/pkg/signature"

params := map[string]string{"amount": "100", "account": "6222****1234"}
sign, timestamp, nonce := signature.GenerateWithNonce(params, secret)
// 设置 Header:
//   X-Signature: {sign}
//   X-Timestamp: {timestamp}
//   X-Nonce: {nonce}
```

### 服务端路由

```go
// 需要验签的路由组
signed := v1.Group("/signed")
signed.Use(middleware.Signature(cfg.Signature, rdb))
signed.POST("/withdraw/apply", withdrawH.Apply)
```
